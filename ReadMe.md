    <h2>📌 convention</h2>
  </summary>
    <h3>commit message</h3>

```
core: 패키지 설정
feat: 신규 기능
test: 테스트 코드
refactor: 리팩터링
docs: 문서 작성
style: 스타일링
fix: 오류 수정
opt: 최적화
```

<hr>

<h3>
  branch
</h3>

```
  label / author / main
```

ex) `feat` / `sg` / `set-up-project`

<hr>

<h3>
  interface & model
</h3>

```
`~Schema` : API response 모델

`~Props` : 컴포넌트 인터페이스

`~Type` : client-side 데이터 모델

상수: UPPER_CASE
컴포넌트, 타입: PascalCase
ETC: camelCase
```
